Create forms so end-users can interact with your application.

A form can be created with a right-click on the "Forms" node inside your solution. You can also do a left-click on that node and click the icon in the detail panel of the Solution Explorer.

Form name

Choose a form name that makes sense to you. Also be aware that the amount of forms can significantly grow over time.

Datasource

You can bind an SQL-based table (or view) to a form. You can use any table from defined servers (server is a connection to an SQL database or in mem database). By doing this, you can select its columns, calculations and aggregations as a "data provider" on form elements, so that record values are automatically displayed and synchronized between client and server.

Extends

Select another form if you want to extend (or inherit) properties of another form.

Working Set

Choose an existing working set where you want your form to be added. The working set is a logical grouping of forms for better organization. A working set can be created from context menu of Forms node in Solution Explorer

Type

Form type specifies the User Interface of the form. Abstract form can only contain code, will be used for common functionality in form hierarchy. CSS Position is the form type in NGClient, uses css top, right, bottom, left, width and height attributes. Responsive form is more complex NGClient form which uses Bootstrap 12-grid layout(by default).

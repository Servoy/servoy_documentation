Here you can create a new module for your solution.

The module (Eclipse project) contains User Interface screens (forms) and code you want to run into your application.

Solution name

Choose a name for your module that makes sense.


Advanced Settings

Solution type

The solution type determines whether your module is a utility solution which contains common functionality with UI (NG Module) or without ui (Module), code to be execute at import (pre or post import

bold> module) or authentication code through a authenticator module.

Learn more in our Wiki here

Location

By default the module project will be created in the folder of the current workspace. You can also create the module in a different location, such as your git folder.

Resources project

You can choose whether your new module will make use of the resources, such as database servers, styles and security. You can choose to have the same resources project as the main solution or no resource project.

In case you are creating a more dynamic module with utility methods, it is usually better to choose no resources module. This means that you are able to use it in other projects as well without the dependency on these particular resources.
